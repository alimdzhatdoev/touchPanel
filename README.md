# Touch Panel Studio (MVP)

Локальное автономное desktop-приложение для Windows touch panels: Admin Studio (редактор) + Runtime Player (воспроизведение).

## Что это умеет (кратко)

- **Проекты**: создание/дублирование/архивирование, импорт/экспорт.
- **Экраны**: несколько экранов в проекте, настройка размера и фона.
- **Компоненты**: текст, кнопка, изображение, фигуры (прямоугольник/эллипс/линия).
- **WYSIWYG**: runtime старается совпадать с тем, что вы видите на канвасе редактора.
- **Пользователи**: вход по логину/паролю, админ может создавать другие учётки.

## Требования

- Windows 10/11 (x64)
- **Python 3.10–3.12** с установкой в PATH ([скачать](https://www.python.org/downloads/windows/)) — при установке включите **«Add python.exe to PATH»**

Целевой стек для production-сборки — Python 3.12; для разработки достаточно любого из перечисленных.

## Быстрый старт (самый простой способ)

1. Открой папку проекта в проводнике.
2. Дважды щёлкни **`run.bat`**.

При первом запуске скрипт сам создаст `.venv`, поставит зависимости из `requirements.txt` и запустит приложение. Дальше достаточно снова запускать **`run.bat`**.

Если `run.bat` пишет, что Python не найден — поставь Python по ссылке выше и перезапусти.

## Вход и пользователи

- **По умолчанию** (если база пользователей пустая) создаётся пользователь: **логин `admin`, пароль `admin`**.
- **Минимальная длина пароля**: **5** символов.
- На экране **«Проекты»**:
  - **«Мой профиль»** — смена логина/пароля текущего пользователя (требуется текущий пароль).
  - **«Пользователи»** — только для роли **admin**: создать/изменить учётки (роль/активность/пароль).

## Как пользоваться (основной сценарий)

1. Запустите приложение и войдите (например, `admin/admin` при первом запуске).
2. В окне **«Проекты»** создайте проект или откройте существующий.
3. В **Studio**:
   - Слева выберите/создайте экран.
   - Настройте **размер экрана** (по умолчанию 1920×1080) и фон.
   - На канвасе добавляйте компоненты (Текст/Кнопка/Картинка/Фигуры).
   - Выделите элемент → справа меняйте свойства: цвет, обводка, шрифт, выравнивание, «Регистр букв» и т.д.
   - Размер компонента меняется ручками: **углы** и **середины сторон** (горизонтально/вертикально).
4. Нажмите **▶ Runtime** — откроется проигрыватель текущего проекта.

## Где хранятся данные (БД и картинки)

Приложение хранит данные **не рядом с exe**, а в профиле текущего пользователя Windows.

- **Глобальная БД приложения (пользователи/настройки)**:
  - `%LOCALAPPDATA%\TouchPanelStudio\runtime\app.sqlite3`
- **Проекты**:
  - корень: `%LOCALAPPDATA%\TouchPanelStudio\projects\`
  - **БД проекта**: `%LOCALAPPDATA%\TouchPanelStudio\projects\<project_code>\project.sqlite3`
  - **картинки проекта (assets)**: `%LOCALAPPDATA%\TouchPanelStudio\projects\<project_code>\assets\...`

## Запуск из консоли (вручную)

```powershell
cd D:\GitHub\touchPanel
py -m venv .venv
.\.venv\Scripts\python.exe -m pip install -U pip
.\.venv\Scripts\python.exe -m pip install -r requirements.txt
.\.venv\Scripts\python.exe -m touch_panel_studio.app.main
```

Если команды `py` нет, используйте `python` вместо `py`.

Базы данных и логи создаются при первом запуске в `%LOCALAPPDATA%\TouchPanelStudio\`.

## Сборка .exe (PyInstaller)

Основной режим: **onedir** (быстрый старт, меньше проблем с Qt DLL).

```powershell
.\build.bat
```

Результат: `dist\touch_panel_studio\touch_panel_studio.exe` и рядом зависимости.

Дополнительно — **onefile** (один `.exe`, старт медленнее из‑за распаковки):

```powershell
.\.venv\Scripts\python.exe -m PyInstaller --noconfirm --clean touch_panel_studio_onefile.spec
```

Результат: `dist\touch_panel_studio.exe`.

### Иконка приложения (.exe)

- Onefile: иконка берётся из `app.ico` (см. `touch_panel_studio_onefile.spec`).
- Onedir: положите `app.ico` в корень репозитория и раскомментируйте строку `icon=...` в `touch_panel_studio.spec`.

Данные приложения (БД, проекты, логи) в frozen-сборке по-прежнему в `%LOCALAPPDATA%\TouchPanelStudio\`, не рядом с `.exe`.

## Структура

Код организован по слоям: `ui/` (Qt), `application/` (use-cases), `domain/` (правила), `db/` (ORM), `infrastructure/` (файлы/import-export/backup), `core/` (конфиг/логирование/безопасность).
